Variable descriptions#

const int X 🔗 Source

Returns the X-coordinate of a combo from a combodata script, from the ‘this->’ pointer as ‘this->X’.

Using this from any other combodata variable is illegal and undefined.

const int Y 🔗 Source

Returns the Y-coordinate of a combo from a combodata script, from the ‘this->’ pointer as ‘this->Y’.

Using this from any other combodata variable is illegal and undefined.

const int Pos 🔗 Source

Returns the combo screen index of a combo from a combodata script, from the ‘this->’ pointer as ‘this->Pos’.

Using this from any other combodata variable is illegal and undefined.

const int Layer 🔗 Source

Returns the layer the combo is on in a combodata script, from the ‘this->’ pointer as ‘this->Layer’.

Using this from any other combodata variable is illegal and undefined.

untyped[] InitD 🔗 Source

The 2 InitD values for the combo script.

const int ID 🔗 Source

The combo ID this combodata points to.

int OriginalTile 🔗 Source

The ‘Original Tile’ of the combo, used as the start of its animation.

int Frame 🔗 Source

The current animation frame the combo is on in its animation. Set to 0 to reset to start of animation.

int AClk 🔗 Source

The animation clock of the combo. Set to 0 to reset the timer.

int Tile 🔗 Source

The tile currently being displayed by the combo. This is automatically overwritten each frame by the engine.

int Script 🔗 Source

The combo script that this combo runs.

int ASpeed 🔗 Source

The combo animation speed.

int Flip 🔗 Source

value TransformationType (FLIP_ or ROT_)

How to flip (or rotate) the combo.

int Walk 🔗 Source

value ComboSolidityBitflags (CMB_)

The solidity bitmask of the combo (same as Screen->ComboS[]).

int Effect 🔗 Source

The effect bitmask of the combo (same as Screen->ComboE[]).

int Type 🔗 Source

int CSet 🔗 Source

Deprecated: Use CSet2 instead!

int CSet2 🔗 Source

The CSet2 offset for this combo.
